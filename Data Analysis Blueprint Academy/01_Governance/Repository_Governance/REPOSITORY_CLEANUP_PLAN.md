# Repository Cleanup Plan

## Immediate actions

1. Add a root README and governance documents.
2. Keep the current repository private while content access is reviewed.
3. Classify materials as private, public-candidate, or instructor-restricted.
4. Remove unexplained duplicate files from active folders.
5. Retain archive copies only when they have clear version and supersession metadata.
6. Introduce a file-naming and document-code standard.
7. Configure Git LFS for large binary files if the repository will continue storing DOCX, PDF, XLSX, and images.
8. Add `.gitignore` and `.gitattributes`.
9. Protect the `main` branch.
10. Require pull requests for controlled academic and database changes.

## Recommended repository split

### Public repository

- Academy overview
- Curriculum summary
- Sample datasets
- Student lab guides
- Public documentation
- Selected portfolio projects

### Private academic repository

- Instructor answer keys
- Full assessments
- Marking schemes
- Moderation records
- Student records
- Proprietary academy operating documents

## Important warning

The repository is currently private. Instructor answer keys and restricted assessment material must be separated before any public repository or public release is created.
