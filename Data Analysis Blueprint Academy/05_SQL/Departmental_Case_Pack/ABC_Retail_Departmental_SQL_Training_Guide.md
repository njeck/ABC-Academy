# ABC Retail Ltd Departmental SQL Case Pack

**Document Code:** DABA-ABC-SQL-CASE-003
**Version:** 1.0
**Enterprise Simulation:** ABC Retail Ltd

## Purpose

This package transforms the 22 Departmental Analytics Casebook scenarios into a complete SQL Server practical program. It gives students realistic business questions and gives instructors validated reference-query structures.

## Package structure

1. **Student Challenges**
   - 22 departmental cases.
   - 66 business questions.
   - No answers.
   - Evidence and validation prompts.

2. **Instructor Solutions**
   - 66 reference SQL Server solutions.
   - Descriptive, diagnostic, and prescriptive outputs.
   - Uses Phase 1 and Phase 2 enterprise tables.
   - Restricted academic resource.

3. **Case Matrix**
   - Case and question register.
   - Difficulty, marks, technique, required tables, status, and assessment tracking.
   - Instructor review and evidence logs.
   - Dashboard for delivery status.

## Student submission requirements

For every assigned question, the student should submit:

1. Business question.
2. SQL query.
3. Result extract.
4. Validation method.
5. Finding.
6. Recommendation where required.
7. Assumptions and limitations.
8. File or screenshot reference.

## Instructor assessment guidance

Suggested scoring for each five-mark question:

- Correct tables, joins, and grain: 1 mark
- Correct filters and calculations: 2 marks
- Validation evidence: 1 mark
- Clear business interpretation: 1 mark

## Important controls

- Keep the instructor solution file outside the student-visible public repository.
- Solutions are reference approaches; alternate correct queries should receive credit.
- Do not accept results that the learner cannot explain.
- Require validation for percentages, variances, rankings, and prescriptive classifications.
- Queries using `GETDATE()` may produce different overdue classifications over time.
- All data is synthetic and educational.
